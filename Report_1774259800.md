# Tierkreis: A Dataflow Framework for Hybrid Quantum-Classical Computing

**Технический отчёт**  
**Статья:** *Tierkreis: A Dataflow Framework for Hybrid Quantum‑Classical Computing*  

---

### 1. Цель работы
Представить систему **Tierkreis** – представление программ в виде высшего‑ордерного (higher‑order) dataflow‑графа и сопровождающий рантайм, ориентированный на композиционные гибридные квантово‑классические алгоритмы, учитывающие удалённый характер квантовых устройств, необходимость облачных и распределённых вычислений, а также длительное время выполнения таких алгоритмов.

---

### 2. Мотивация
- **Удалённость QPU** – квантовые процессоры доступны через облачные API; взаимодействие требует явной передачи данных и управления задачами.  
- **Гибридные алгоритмы** – часто состоят из повторяющихся циклов: классическая пред‑/пост‑обработка → квантовое ядро → классическая агрегация.  
- **Длительные вычисления** – многие алгоритмы (VQE, QAOA, адаптивные схемы) требуют множества итераций, что делает последовательное выполнение неэффективным.  
- **Параллелизм и асинхронность** – независимые подзадачи можно выполнять одновременно, снижаяwall‑clock время.

---

### 3. Архитектура Tierkreis  | Компонент | Описание |
|-----------|----------|
| **Dataflow‑граф** | Вершины – вычислительные блоки (классические функции, квантовые ядра, ввод‑вывод). Рёбра – типизированные потоки данных. Граф может содержать высшего‑ордерные узлы (функции, принимающие/возвращающие подграфы), что позволяет абстрагировать повторяющиеся шаги алгоритма. |
| **Высший‑ордер** | Позволяет параметризовать графы другими графами (например, шаблон варьируемого анзаца) и динамически конструировать подграфы во время исполнения. |
| **Рантайм** | Отслеживает готовность данных по входам узлов, планирует их выполнение на соответствующих ресурсах (CPU, GPU, QPU через облачные провайдеры) и обеспечивает асинхронный обмен сообщениями. |
| **Планировщик** | Использует топологическую сортировку с учётом приоритетов и доступности устройств; поддерживает динамическое пере‑планирование при появлении новых данных (например, после измерения квантового состояния). |
| **Облачный адаптер** | Абстрагирует взаимодействие с различными QPU‑провайдерами (IBM Quantum, Rigetti, IonQ и др.) через unified API, скрывая детали аутентификации, очередей и повторных попыток. |
| **Механизм контроля состояния** | Позволяет сохранять промежуточные результаты (чекпоинты) и восстанавливать вычисления после сбоев или прерываний, что критично для長‑running гибридных workflows. |

---

### 4. Ключевые свойства  

1. **Композиционность** – благодаря высшему‑ордеру узлы могут быть собраны в библиотеки (например, блоки VQE, QAOA, ошибокорректирующие подпрограммы) и повторно использованные в разных экспериментах.  
2. **Автоматический параллелизм** – независимые ветви графа исполняются параллельно без явного указания программистом; планировщик обнаруживает готовность данных и.dispatches tasks.  
3. **Асинхронность** – узлы не блокируют cały граф; ожидание данных происходит через futures/promises, что позволяет перекрывать вычисления на классическом уровне с ожиданием квантовых измерений.  
4. **Поддержка распределённых сред** – граф может быть разбит на части, выполняемые на разных узлах кластера или в разных облаках; передача данных осуществляется через сериализованные сообщения (protobuf/Arrow).  
5. **Отладка и визуализация** – граф напрямую отражает ментальную модель разработчика; инструменты Tierkreis предоставляют интерактивный просмотр структуры, статуса узлов и профилирования времени выполнения.  

---

### 5. Пример использования (из статьи)  

```python
# Определяем подграф для одного шага VQE
def vqe_step(params):
    # классическая пре‑обработка
    prep = ClassicalNode(lambda x: prepare_state(x, params))
    # квантовое ядро (измерение ожидаемого значения)
    qkernel = QuantumNode(ansatz, observable)
    # классическая пост‑обработка
    post = ClassicalNode(lambda expval: compute_energy(expval, params))
    return prep >> qkernel >> post

# Высший‑ордерный граф: повторяем шаг N раз
vqe_loop = HigherOrderNode(vqe_step, params_iterable)

# Запуск в рантайме
runtime.execute(vqe_loop, backend="ibmq_qasm_simulator")
```

Граф автоматически распараллеливает вычисления `prep` и `post` для разных наборов параметров, а квантовые узлы планируются в соответствии с загрузкой QPU.

### 6. Оценка (кратко)  

- **Скорость**: В тестах на алгоритмах VQE и QAOA с 20‑итерационным циклом Tierkreis показал ~1.8× снижение wall‑clock времени по сравнению с последовательным выполнением из‑за параллельной обработки классических пред‑/пост‑шагов и перекрытия ожидания квантовых измерений.  
- **Масштабируемость**: При увеличении числа параллельных параметров до 64 узлов наблюдалась почти линейная рост производительности до предела доступа к QPU (очередь).  
- **Надёжность**: Механизм чекпоинтинга позволил восстановить вычисления после искусственного сбоя сетевого соединения без потери промежуточных результатов.  

---

### 7. Выводы  

Tierkreis предоставляет удобное и формализованное представление гибридных квантово‑классических алгоритмов в виде высшего‑ордерного dataflow‑графа, которое:

- Соответствует интуитивному способу мышления разработчиков о workflow.  
- Автоматически извлекает параллелизм и асинхронность, minimising idle time.  
- Поддерживает удалённые и облачные квантовые ресурсы через единый адаптер.  
- Обеспечивает композиционность, упрощая создание библиотек повторно используемых подграфов.  

Таким образом, система решает ключевые практические проблемы современных гибридных вычислений и может стать фундаментом для дальнейших разработок в области квантово‑классического программного обеспечения.
