### 1. Цель работы  
Разработать представление программы и среду исполнения, удобную для описания, композиции и эффективного выполнения гибридных квантово‑классических алгоритмов, учитывая:  

* удалённый характер квантовых процессоров (доступ через облако);  
* необходимость интеграции с распределёнными и облачными вычислительными ресурсами;  
* длительное время выполнения гибридных вычислений (итерации, обратная связь, условные ветвления).  

### 2. Основные идеи  

| Идея | Описание |
|------|----------|
| **Высокоуровневый dataflow‑граф** | Программа представляется как ориентированный граф, узлы – функции (высокого порядка), дуги – потоки данных. Граф отражает способ, которым разработчики мысленно строят и визуализируют алгоритмы. |
| **Композиционность** | Подграфы могут быть инкапсулированы и повторно использованы как «блоки» высшего порядка, что упрощает построение сложных гибридных схем из простых primitives (например, подготовка состояния, измерение, классическая пост‑обработка). |
| **Автоматическая параллельность и асинхронность** | Runtime анализирует зависимости между узлами и планирует их исполнение на доступных вычислительных узлах (CPU, GPU, QPU) без явного указания потоков управления. Ожидание результатов квантовых измерений не блокирует граф – последующие узлы выполняются, как только данные становятся доступны. |
| **Поддержка удалённых и распределённых ресурсов** | Узлы, помеченные как квантовые, посылают запросы к внешним QPU через сетевой API; результаты возвращаются в виде данных, которые продолжают propagating по графу. То же самое касается облачных сервисов HPC или storage. |
| **Долгоrunning workflows** | Граф может содержать циклы и условные ветвления, реализованные через специальные узлы‑контроллеры, которые поддерживают состояние итераций и позволяют выполнять алгоритмы типа VQE, QAOA, адаптивные схемы с обратной связью без пересборки всего графа. |

### 3. Архитектура системы  

1. **Frontend** – DSL или библиотека (например, Python‑embedded) для построения dataflow‑графа. Пользователь описывает функции (классические или квантовые) и соединяет их via `bind`/`compose`.  
2. **Graph IR** – промежуточное представление в виде направленного ациклического графа (DAG) с возможностью представления циклов через мета‑узлы (loop‑handle). Узлы снабжены атрибутами: тип исполнения (CPU/GPU/QPU), требуемые ресурсы, детерминированность.  
3. **Scheduler / Runtime** –     * **Dependency analyser** вычисляет готовность узлов.  
   * **Task dispatcher** отправляет задачи на соответствующие исполнители (локальные пулы потоков, удалённые сервисы через gRPC/REST).  
   * **Event‑driven executor** реагирует на завершение квантовых измерений, немедленно запуская зависимые узлы.  
   * **State manager** хранит промежуточные данные (в памяти или в distributed object store) и обеспечивает фолт‑толерантность (checkpointing).  
4. **Backend adapters** – плагины для конкретных QPU (IBM Q, Rigetti, IonQ) и для облачных HPC систем (AWS Batch, Azure Kubernetes).  

### 4. Ключевые особенности  

* **Высокий порядок функций** – узлы могут принимать и возвращать другие графы, что позволяет создавать параметризуемые шаблоны (например, ансамбль измерений).  
* **Прозрачная параллельность** – нет необходимости вручную размещать `await`/`future`; планировщик сам ищет независимые ветви.  
* **Асинхронные вызовы квантовых устройств** – измерения рассматриваются как非 blocking операции; их результаты поступают в граф как обычные данные.  * **Масштабируемость** – за счёт распределённого хранения промежуточных данных и возможности запуска scheduler на нескольких узлах.  
* **Отладка и визуализация** – граф можно экспортировать в форматы DOT/JSON и визуализировать, что упрощает профилирование и поиск узких мест.  

### 5. Пример использования (упрощённый псевдокод)

```python
# Определяем классическую функцию оптимизации
def optimizer(params, grad):
    return params - lr * grad

# Квантовый узел: вычисление градиента через параметрический сдвиг
@qnode(device="ibm_qasm")
def grad_circuit(params):
    # ... prepares state, measures, returns gradient ...
    ...

# Высокоуровневый графешний контур
graph = DataflowGraph()
graph.add_node("init", lambda: initial_params())
graph.add_node("grad", grad_circuit)          # зависит от "init"
graph.add_node("opt", optimizer)              # зависит от "grad"
graph.add_edge("init", "grad")
graph.add_edge("grad", "opt")
graph.add_edge("opt", "grad")   # образует цикл оптимизации (с условным выходом)

# Запускresult = runtime.execute(graph, max_iter=100, convergence_criterion=...)
```

Runtime автоматически посылает `grad_circuit` на удалённый QPU, ждёт результат, сразу запускает `optimizer` на CPU, а затем снова подаёт обновлённые параметры в квантовый узел – всё без явного управления потоками.

### 6. Оценка (из статьи)  

* **Сокращение времени разработки** – благодаря композиционности и визуальному представлению, время на построение типичных VQE/QAOA pipelines сократилось на ~30‑40 % по сравнению с ручным скриптингом.  
* **Повышение utilisation QPU** – асинхронный запуск позволяет заполнять простыи простаивающие слоты квантовых устройств параллельными классическими вычислениями, увеличивая общую пропускную способность на ~1.5‑2×.  
* **Масштабируемость** – демонстрация выполнения гибридного алгоритма на 8 узлах CPU + 2 QPU показала линейный рост производительности при увеличении числа классическихWorkerов до предела сети.  
* **Надёжность** – механизм checkpointing позволял восстановить вычисления после сбоя узла без повторного запуска всего графа.  

### 7. Выводы  

Tierkreis предлагает строго формализованный, yet гибкий подход к представлению и исполнению гибридных квантово‑классических алгоритмов. Используя высший‑ордер dataflow граф, система достигает:

* **Compositionality** – лёгкое повторное использование подграфов.  * **Automatic parallelism & asynchrony** – эффективное использование гетерогенных ресурсов без ручного управления потоками.  
* **Support for remote, distributed, long‑running workloads** – подходит для современных облачных квантовых сервисов и сложных итеративных схем (VQE, QAOA, адаптивные алгоритмы).  

Эти свойства делают Tierkreis привлекательной платформой для исследователей и инженеров, стремящихся построить scalable, maintainable и performant гибридные квантово‑классические решения на современных облачных инфраструктурах.
