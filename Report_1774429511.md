### 1. Цель работы  
Разработать систему **Tierkreis** — представление программ в виде высшего‑ордерного потока данных (higher‑order dataflow graph) и соответствующий рантайм, который упрощает:

* композицию гибридных квантово‑классических алгоритмов;  
* работу с удалёнными квантовыми процессорами (облачные QPU);  
* распределённые и долго‑работающие вычисления, требующие автоматической параллелизции и асинхронности.

---

### 2. Ключевые идеи и мотивация  

| Проблема | Как решается в Tierkreis |
|----------|--------------------------|
| **Удалённый доступ к QPU** (задержки, ограниченный доступ) | Граф описывает зависимости между квантовыми и классическими задачами; планировщик может откладывать выполнение квантовых узлов до готовности QPU, а остальные части продолжают работать асинхронно. |
| **Гибридные алгоритмы требуют облачных/распределённых ресурсов** | Данный граф является переносимым представлением; узлы могут быть выполнены локально, в контейнерах, на кластерах или в сервисах FaaS. |
| **Долго‑работающие вычисления** (итерационные схемы, VQE, QAOA и т.п.) | Поддержка циклических графов и состояний (stateful nodes) позволяет сохранять промежуточные результаты между итерациями без полной перезаписи. |
| **Необходимость визуализации и рассуждений о алгоритме** | Граф‑модель напрямую отражает диаграммы, которые обычно рисуют разработчики (входы/выходы, ветвление, редукция). Это упрощает отладку и оптимизацию. |

### 3. Архитектура Tierkreis  

1. **Dataflow Graph (DFG)**     * Узлы (`Node`) – высшего порядка функции, могут принимать в качестве аргументов другие графы (подграфы) или квантовые цепи.  
   * Рёбра – типизированные потоки данных (классические значения, квантовые состояния, измерения, управляющие сигналы).  
   * Поддерживаются:  
     - **Последовательные** зависимости (конвейер).  
     - **Ветвление** (условные узлы, основанные на результатах измерений).       - **Редукция/агрегация** (например, сбор статистики из множества запусков).  
     - **Циклические** подграфы для итерационных алгоритмов.  

2. **Типы узлов**  
   * **ClassicalNode** – произвольный классический код (Python, C++, WASM).  
   * **QuantumNode** – описание квантовой цепи (через OpenQASM, Qiskit, Cirq и т.д.); выполнение делегируется удалённому QPU через абстракцию *quantum backend*.  
   * **ControlNode** – узел, реализующий логику ветвления на основе классических измерений (feed‑forward).     * **Higher‑OrderNode** – принимает подграф как параметр, enabling compositional patterns (например,map‑over‑circuits).  

3. **Runtime / Scheduler**  
   * **Task Graph Executor** – трансформирует DFG в набор готовых к выполнению задач, учитывая доступность ресурсов (локальные CPU/GPU, удалённые QPU, сетевые задержки).  
   * **Асинхронный диспетчер** – задачи запускаются сразу, как только все их входные рёбра готовы; результаты помещаются в типизированный канал (future/promise).  
   * **Планировщик размещения** – выбирает бекенд для каждого QuantumNode (симулятор, реальный процессор, облачный провайдер) на основе политик (стоимость, время ожидания, fidelity).  
   * **Механизм кэширования состояний** – сохраняет промежуточные квантовые состояния (например, после предварительной подготовки) для повторного использования в итерациях.  
   * **Мониторинг и визуализация** – встроенный экспортер в форматы DOT/JSON, интеграция с Jupyter‑notebook и веб‑дашбордом для отслеживания прогресса графов.  

4. **Интерфейс пользователя**  
   * API на Python (декораторы `@node`, `@quantum`, `@classical`) позволяют описывать алгоритмы почти как обычный код, а фреймворк автоматически строит DFG.  
   * Возможность экспортировать/импортировать графы в стандартные форматы (JSON‑LD, GraphML) для межъязыковой совместимости.

### 4. Пример использования (вариационный własный собственный eigen‑solver, VQE)  

```python
@quantum
def ansatz(params):
    # параметризованная цепь
    ...

@classical
def cost_function(counts):
    # вычисление ожидаемого значения энергии
    ...

@node
def vqe_step(params):
    counts = ansatz(params)          # QuantumNode → удалённый QPU    return cost_function(counts)    # ClassicalNode

# граф: итерационный цикл с оптимизатором (ClassicalNode)
opt = Optimizer()
for i in range(max_iter):
    grad = opt.gradient(vqe_step, params_i)
    params_i+1 = opt.update(params_i, grad)
```

*Система автоматически*:  
* отправляет `ansatz` на выбранный QPU;  
* ждёт результата измерения;  
* параллельно вычисляет градиент на CPU;  
* обновляет параметры и запускает следующую итерацию, не блокируя весь пайплайн.

### 6. Выводы  

* Tierkreis предлагает **формальное представление гибридных квантово‑классических алгоритмов** в виде высшего‑ордерного dataflow графа, что делает их явными, композиционными и легко визуализируемыми.  * Встроенный **рантайм с асинхронным планировщиком и поддержкой удалённых QPU** решает практические проблемы: сетевые задержки, heterogeneous resources, длительные итерационные вычисления.  
* Благодаря **автоматическому параллелизму и возможности кэширования промежуточных состояний**, framework достигает ощутимого ускорения без ручного управления потоками выполнения.  
* API на уровне декораторов позволяет исследователям сосредоточиться на самом алгоритме, а не на низкоуровневых деталях распределённых вычислений.  

Таким образом, Tierkreis stanowiт важный шаг к **практической эксплуатации гибридных квантово‑классических вычислений** в облачных и распределённых средах, облегчая как проектирование, так и исполнение сложных алгоритмов.
