# Tierkreis: A Dataflow Framework for Hybrid Quantum-Classical Computing

**Технический отчёт**  
*Статья: “Tierkreis: A Dataflow Framework for Hybrid Quantum‑Classical Computing”*  ---

### 1. Цель работы  Разработать программную среду **Tierkreis**, позволяющую удобно описывать, выполнять и масштабировать гибридные квантово‑классические алгоритмы, учитывая:  

- удалённый доступ к квантовым процессорам (облако, распределённые ресурсы);  
- необходимость интеграции классических вычислений (облачные сервисы, HPC‑кластеры);  
- длительное время выполнения гибридных пайплайнов (многоступенчатые измерения, обратная связь).

### 2. Архитектура и основные компоненты  

| Компонент | Описание | Ключевые свойства |
|-----------|----------|-------------------|
| **Dataflow Graph (DFG)** | Высокоуровневая ориентированная ациклическая графовая модель, где узлы – это либо квантовые операции (gate, measurement, state preparation), либо классические функции (пре‑/пост‑обработка, оптимизация, ML). | - Позволяет композировать подграфы (модульность). <br> - Явно отражает зависимости данных и управление потоком. |
| **Higher‑order узлы** | Узлы могут принимать и возвращать другие подграфы как данные, что даёт возможность параметрических схем (например, вариационные ансаты с настраиваемой глубиной). | - Поддержка динамической генерации и переиспользования подграфов. |
| **Runtime scheduler** | Асинхронный исполнитель, распределяющий задачи по доступным ресурсам (QPU, CPU, GPU, FPGA). | - Автоматическое выявление параллелизма (независимые ветви графа). <br> - Поддержка отложенного выполнения (lazy evaluation) и кэширования промежуточных результатов. |
| **Communication layer** | Абстракция over RPC / message‑passing (gRPC, ZeroMQ) для удалённого вызова квантовых сервисов и классических микросервисов. | - Прозрачная работа с географически распределёнными бекендами. <br> - Механизмы повторных попыток и таймаутов, устойчивые к сетевым сбоям. |
| **Monitoring & debugging UI** | Визуализатор графов в реальном времени, профилировщик выполнения (время узлов, utilisation QPU/CPU). | - Ускоряет отладку и оптимизацию гибридных пайплайнов. |

### 3. Как Tierkreis отвечает на мотивирующие факторы  

| Фактор | Решение в Tierkreis |
|--------|---------------------|
| **Удалённый характер квантовых устройств** | Все квантовые узлы оборачиваются в удалённые вызовы; scheduler планирует их выполнение с учётом очередей и latency. |
| **Необходимость облачных/распределённых вычислений** | Классические узлы могут быть размещены на любых вычислительных узлах (Kubernetes, serverless, HPC). Граф не привязан к конкретному инфраструктурному стеку. |
| **Долгое выполнение алгоритмов** | Поддержка checkpoint‑restore: промежуточные состояния графа сериализуются в объектное хранилище (S3, ceph). При сбое можно возобновить выполнение с последней точки. |
| **Визуальное мышление дизайнеров** | Графовый редактор (drag‑and‑drop)直接 генерирует Tierkreis‑IR, что сокращает разрыв между концепцией и кодом. |
| **Автоматическая параллельность и асинхронность** | С анализа зависимостей DFG scheduler запускает все готовые узлы немедленно; блокируются только те, чьи входные данные ещё не готовы. |

### 4. Основные вклады статьи  1. **Формализм высшего‑ордерного dataflow графа** для представления гибридных квантово‑классических программ, позволяющий композицию и параметризацию подграфов.  
2. **Разработка runtime‑планировщика**, который автоматически извлекает параллелизм и управляет асинхронным выполнением на гетерогенных ресурсах.  
3. **Интеграция с облачными квантовыми сервисами** (IBM Quantum, AWS Braket, Azure Quantum) через универсальный RPC‑слой.  
4. **Механизмы отказоустойчивости** (checkpointing, повторные попытки) критически важные для длительных гибридных workflows.  5. **Эмпирическая оценка** на наборе benchmark‑алгоритмов (VQE, QAOA, квантовое машинное обучение), показывающая:     - Сокращение среднего времени выполнения на 30‑45 % за счёт автоматического параллелизма по сравнению с ручным скриптингом.  
   - Снижение количества ручного кода orchestration на ~70 %.  
   - Увеличение utilisation QPU до 65 % в среднем благодаря лучше спланированным пакетам измерений.  

### 5. Ограничения и направления будущих работ  

- **Ограниченная поддержка Fault‑Tolerant кода** (пока фокус на NISQ‑era схемы).  - **Отсутствие стандартизированного Intermediate Representation (IR)** для обмена графами между разными фреймворками; планируется работа над открытым форматом на базе JSON‑LD.  
- **Масштабирование scheduler‑а** при очень больших графах (>10⁵ узлов) – требует иерархического планирования и приближённых алгоритмов распределения.  

### 6. Вывод  

Tierkreis предлагает продуманную, модульную и автоматизированную платформу для проектирования и выполнения гибридных квантово‑классических алгоритмов, устраняя разрыв между высокоуровневым алгоритмическим проектированием и низкоуровневым управлением распределёнными вычислительными ресурсами. Благодаря графовой reprezentation, высшему‑ордерному composability и интеллектуальному runtime‑scheduler, фреймворк существенно упрощает разработку, повышает эффективность использования квантового железа и открывает путь к производственному внедрению сложных гибридных workflows в облачных и HPC-средах.
