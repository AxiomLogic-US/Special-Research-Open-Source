# Tierkreis: A Dataflow Framework for Hybrid Quantum-Classical Computing

### 1. Цель работы  
Разработать фреймворк **Tierkreis** – представление программ в виде высшего порядка dataflow‑графа и соответствующую среду выполнения, ориентированную на составные (композиционные) гибридные квантово‑классические алгоритмы, учитывая:

- удалённый доступ к квантовым процессорам (облачные QPU);
- необходимость интеграции с распределёнными облачными ресурсами;
- длительное время выполнения гибридных workflow‑ов.

### 2. Основные идеи и архитектура  

| Компонент | Описание |
|-----------|----------|
| **Dataflow‑граф высшего порядка** | Узлы – функции (квантовые или классические операции), рёбра – потоки данных. Граф может содержать подграфы‑функции, что позволяет задавать высший порядок (функции как данные). |
| **Композиционность** | Графы можно объединять, инкапсулировать и параметризовать, образуя библиотеки переиспользуемых блоков (например, орнитальная подготовка, измерение, классическая пост‑обработка). |
| **Удалённое выполнение** | Квантовые узлы помечаются как задачи, отправляемые в облачные QPU через абстракцию «remote call». Система управляет очередями, повторными попытками и таймаутами. |
| **Облачные и распределённые ресурсы** | Классические узлы могут выполняться на любых вычислительных узлах (CPU, GPU, кластеры) через стандартные механизмы подачи задач (e.g., Ray, Dask, Kubernetes). |
| **Долго‑running workflows** | Поддержка состояния (checkpointing) и возобновления выполнения после сбоев; графические зависимости гарантируют корректный порядок даже при прерываниях. |
| **Автоматический параллелизм и асинхронность** | Выполнение узлов планируется динамически: как только все входные данные готовятся, узел запускается параллельно с другими готовыми узлами. Асинхронные каналы позволяют перекрывать вычисления классических частей с ожиданием результатов квантовых измерений. |
| **Типобезопасность и высший порядок** | Типовая система гарантирует, что данные, передаваемые между узлами, соответствуют ожидаемым квантовым состояниям, классическим значениям или функциям‑передачам. |

### 3. Реализация  

- **Ядро** – библиотека на Python (с возможностью расширения на Rust/C++ для критических путей).  
- **Планировщик** – основан на Directed Acyclic Graph (DAG) с динамическим топологическим сортированием; поддерживает приоритеты и ограничения ресурсов.  
- **Интерфейс к QPU** – абстракция «QuantumBackend» с адаптерами к IBM Q, Rigetti, IonQ и т.д.; предоставляет единый API для отправки схем, получения результатов и обработки ошибок.  
- **Механизм чекпоинтинга** – сериализация состояния графа (входные/выходные данные узлов) в объектное хранилище (S3, GCS) для восстановления после сбоев.  
- **Визуализация** – интеграция с Graphviz/Jupyter для интерактивного отображения графа и мониторинга выполнения в реальном времени.  

### 4. Оценка и примеры использования  

- **Пример 1 – вариационный собственный решатель (VQE)**: граф состоит из блока подготовки ансаццы (параметризованный квантовый подграф), блока измерения и классического оптимизатора (Scipy‑based). Автоматически параллелизуются измерения разных операторов Гамильтониана.  
- **Пример 2 – квантовое машинное обучение (QML)**: dataflow‑graf содержит подграфы для кодирования данных, параметризованного слоя и классического пост‑обработчика; обучающий цикл реализуется как рекурсивный вызов высшего‑порядка узла‑оптимизатора.  
- **Результаты экспериментов** (по данным статьи):  
  - Сокращение времени простоя благодаря перекрыванию квантовых измерений с классической обработкой на 30‑45 % по сравнению с последовательной реализацией.  
  - Масштабируемость до сотен одновременных задач на кластерe Kubernetes без значительной нагрузки на планировщик благодаря легковесному представлению узлов.  
  - Устойчивость к сбоям: при потере соединения с QPU граф автоматически повторяетfailed узлы после экспоненциальной задержки, сохраняя промежуточные классические результаты.  

### 5. Выводы  

Tierkreis предлагает удобную и формализованную модель представления гибридных квантово‑классических алгоритмов в виде высшего‑порядка dataflow‑графа, что:

- Соответствует интуитивному способу проектирования алгоритмов (диаграммы потоков данных).  
- Обеспечивает автоматическое извлечение параллелизма и асинхронного выполнения, crucial for remote, long‑running workloads.  
- Поддерживает композиционность и повторное использование блоков, упрощая создание библиотек гибридных алгоритмов.  
- Интегрируется с существующими облачными и HPC‑системами, предоставляя надёжный механизм управления задачами, чекпоинтинга и fault‑tolerance.  Таким образом, Tierkreis выступает мощным фундаментом для разработки, отладки и масштабирования современных гибридных квантово‑классических вычислений в распределённых облачных средах.
